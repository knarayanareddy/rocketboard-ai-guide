1. CORE ENTITY TABLES
----------------------

TABLE: orgs
  Columns: id UUID PK, name TEXT, created_at TIMESTAMPTZ
  Purpose: Organizations (top-level tenants)

TABLE: packs
  Columns: id UUID PK, org_id UUID FK orgs, title TEXT, description TEXT,
           language_mode TEXT, grounding_policy JSONB,
           roadmap_enabled BOOLEAN DEFAULT false,
           created_by UUID FK auth.users, created_at TIMESTAMPTZ, updated_at TIMESTAMPTZ
  Purpose: Knowledge packs (the unit of isolation)

TABLE: pack_members
  Columns: id UUID PK, pack_id UUID FK packs, user_id UUID FK auth.users,
           role TEXT ('learner'|'author'|'admin'), created_at TIMESTAMPTZ
  Purpose: Pack membership and role assignments
  RLS:
    - SELECT: pack member can see own row; authors see all rows for their pack
    - INSERT: authors can add members
    - DELETE: authors can remove members

TABLE: org_members
  Columns: id UUID PK, org_id UUID FK orgs, user_id UUID FK auth.users,
           role TEXT, created_at TIMESTAMPTZ
  Purpose: Org-level membership

TABLE: profiles
  Columns: id UUID PK (= auth.uid()), display_name TEXT, avatar_url TEXT,
           created_at TIMESTAMPTZ, updated_at TIMESTAMPTZ
  RLS: users can read/update own profile only

TABLE: pack_sources
  Columns: id UUID PK, pack_id UUID FK packs, source_type TEXT, label TEXT,
           source_uri TEXT, config JSONB (non-secret settings only),
           last_synced_at TIMESTAMPTZ, created_at TIMESTAMPTZ
  Purpose: Tracks connected data sources per pack
  Security: Vault credentials stored separately, never in config

2. KNOWLEDGE CHUNKS (Core RAG Table)
--------------------------------------

TABLE: knowledge_chunks
  Columns:
    id UUID PK DEFAULT gen_random_uuid()
    pack_id UUID FK packs (NOT NULL)
    source_id UUID FK pack_sources (NOT NULL)
    org_id UUID (NOT NULL)
    chunk_id TEXT (NOT NULL)        -- e.g., 'C00001' (per-ingestion run ID)
    path TEXT (NOT NULL)             -- e.g., 'repo:owner/name/src/foo.ts'
    content TEXT                     -- chunk text (may be redacted inline)
    embedding VECTOR(1536)           -- OpenAI text-embedding-3-small
    fts TSVECTOR                     -- full-text search index (auto-generated)
    line_start INT
    line_end INT
    entity_type TEXT                 -- 'function' | 'class' | 'interface' | etc.
    entity_name TEXT                 -- e.g., 'MyClass' or 'handleChat'
    signature TEXT                   -- function/type signature string
    imports TEXT[]                   -- array of imported identifiers
    exported_names TEXT[]            -- array of exported identifiers
    content_hash TEXT                -- SHA256 of content (for deduplication/reuse)
    is_redacted BOOLEAN DEFAULT false -- true = chunk excluded from retrieval
    module_key TEXT                  -- optional context filter
    track_key TEXT                   -- optional context filter
    generation_id UUID               -- links to active generation for the pack
    ingestion_job_id UUID FK ingestion_jobs
    metadata JSONB                   -- source_url, redaction info, setup category
    created_at TIMESTAMPTZ DEFAULT now()
    updated_at TIMESTAMPTZ

  UNIQUE CONSTRAINT: (pack_id, chunk_id)

  RLS: knowledge_chunks has RLS enabled but retrieval uses service_role via
       Edge Functions; direct client queries are blocked for most roles.
       Pack members can SELECT their own pack's non-redacted chunks
       (check migrations for latest policy).

  INDEXES:
    idx on (pack_id, source_id)      -- for purge operations
    idx on (org_id, pack_id)         -- for retrieval scoping
    HNSW index on embedding          -- for pgvector cosine similarity
    GIN index on fts                 -- for full-text search
    idx on (pack_id, entity_name)    -- for definition_search_v1
    idx on exports + imports         -- for graph expansion in hybrid_search_v2

3. INGESTION TABLES
--------------------

TABLE: ingestion_jobs
  Columns: id UUID PK, pack_id UUID FK packs, source_id UUID FK pack_sources,
           status TEXT ('processing'|'completed'|'failed'),
           started_at TIMESTAMPTZ, completed_at TIMESTAMPTZ,
           total_chunks INT, processed_chunks INT,
           retry_count INT DEFAULT 0, last_error_at TIMESTAMPTZ,
           last_error_message TEXT, error_message TEXT,
           metadata JSONB, created_at TIMESTAMPTZ

TABLE: pack_active_generation
  Columns: org_id UUID, pack_id UUID, active_generation_id UUID,
           updated_at TIMESTAMPTZ
  Purpose: Tracks the currently active chunk generation per pack.
           hybrid_search_v2 uses this to filter stale chunks.

4. RAG METRICS AND OBSERVABILITY
----------------------------------

TABLE: rag_metrics
  Columns: id UUID PK, org_id UUID, user_id UUID, query TEXT, task_type TEXT,
           request_id TEXT, retrieval_method TEXT, chunks_retrieved INT,
           chunks_after_rerank INT, avg_relevance_score FLOAT,
           retrieval_latency_ms INT, model_used TEXT, provider_used TEXT,
           generation_latency_ms INT, input_tokens INT, output_tokens INT,
           citations_found INT, citations_verified INT, citations_failed INT,
           grounding_score FLOAT, verification_score FLOAT, strip_rate FLOAT,
           grounding_gate_passed BOOLEAN, grounding_gate_reason TEXT,
           attempts INT DEFAULT 1, agent_confidence TEXT,
           total_latency_ms INT, pack_id UUID, trace_id TEXT,
           created_at TIMESTAMPTZ DEFAULT now()
  RLS:
    SELECT: org members can view metrics for their org
    INSERT: service role only (no policy = blocked for authenticated)

TABLE: pack_quality_daily (rollup table)
  Columns: pack_id UUID, day DATE,
           total_requests INT, passed INT, refused INT, retried INT,
           avg_strip_rate NUMERIC, avg_citations_found NUMERIC,
           avg_attempts NUMERIC, total_latency_ms BIGINT, p95_latency_ms INT
  Unique constraints: (pack_id, day)
  RLS: pack authors/admins can SELECT

5. AI AUDIT LOG
----------------

TABLE: ai_audit_events
  Columns: id UUID PK, created_at TIMESTAMPTZ,
           org_id UUID NULL, pack_id UUID FK packs, user_id UUID FK auth.users,
           task_type TEXT, request_id TEXT, trace_id TEXT, provider_used TEXT,
           model_used TEXT, grounding_gate_passed BOOLEAN,
           grounding_gate_reason TEXT, attempts INT, strip_rate NUMERIC(6,4),
           citations_found INT, unique_files_count INT,
           evidence_manifest JSONB (citations array, spans array),
           prompt_hash TEXT, response_hash TEXT,
           prompt_preview TEXT, response_preview TEXT
  RLS:
    SELECT: pack authors/admins only (has_pack_access(uid, pack_id, 'author'))
    INSERT: service role only
    No UPDATE or DELETE policies (append-only via RLS)
  INDEXES:
    idx on (pack_id, created_at DESC)
    idx on (request_id)
    idx on (user_id, created_at DESC)

6. LIFECYCLE TABLES
--------------------

TABLE: pack_lifecycle_policies
  Columns: pack_id UUID PK FK packs,
           retention_rag_metrics_days INT DEFAULT 90,
           retention_ingestion_jobs_days INT DEFAULT 90,
           retention_audit_days INT DEFAULT 365 (added in ai_audit_events migration),
           legal_hold BOOLEAN DEFAULT false,
           updated_by UUID FK auth.users, updated_at TIMESTAMPTZ
  RLS: pack authors/admins only (USING + WITH CHECK via has_pack_access)

TABLE: lifecycle_audit_events
  Columns: id UUID PK, pack_id UUID FK packs, actor_user_id UUID,
           action TEXT ('purge_source'|'retention_cleanup'),
           target_type TEXT ('source'|'table'),
           target_id UUID, parameters JSONB, rows_deleted JSONB,
           status TEXT ('started'|'completed'|'failed'),
           error_message TEXT, created_at TIMESTAMPTZ
  RLS: pack authors/admins can SELECT only

7. ROADMAP / PLAYLIST TABLES
------------------------------

TABLE: playlists
  Columns: id UUID PK, pack_id UUID FK packs, title TEXT, description TEXT,
           phase TEXT CHECK ('day_1_30'|'day_31_60'|'day_61_90'),
           required BOOLEAN, owner_user_id UUID, owner_display_name TEXT,
           default_start_offset_days INT, created_by UUID,
           created_at TIMESTAMPTZ, updated_at TIMESTAMPTZ
  RLS:
    SELECT: pack members
    ALL: pack authors/admins

TABLE: playlist_items
  Columns: id UUID PK, playlist_id UUID FK playlists, sort_order INT,
           title TEXT, description TEXT, required BOOLEAN,
           item_type TEXT CHECK ('module'|'section'|'quiz'|'milestone'|'task'|'link'),
           module_id UUID FK generated_modules, module_key TEXT, section_id TEXT,
           quiz_id UUID, milestone_id UUID FK onboarding_milestones,
           url TEXT, due_offset_days INT, unlock_offset_days INT
  RLS: SELECT by pack members; ALL by pack authors

TABLE: playlist_item_dependencies
  Columns: (item_id, depends_on_item_id) PK, CONSTRAINT no_self_dependency
  Purpose: Directed dependency graph to control item ordering
  RLS: SELECT by pack members; ALL by pack authors

TABLE: playlist_assignments
  Columns: id UUID PK, pack_id UUID, playlist_id UUID, learner_user_id UUID,
           assigned_by UUID, owner_user_id UUID, start_date DATE,
           status TEXT ('active'|'paused'|'completed'|'cancelled'),
           UNIQUE (playlist_id, learner_user_id)
  RLS: learners see own; authors manage all

TABLE: playlist_item_progress
  Columns: id UUID PK, assignment_id UUID FK playlist_assignments,
           item_id UUID FK playlist_items, learner_user_id UUID,
           status TEXT ('blocked'|'available'|'in_progress'|'done'|'skipped'),
           started_at TIMESTAMPTZ, completed_at TIMESTAMPTZ,
           last_event_at TIMESTAMPTZ, note TEXT
  RLS: learners manage own; authors view all

VIEW: view_playlist_item_state
  Joins assignments + items + progress to compute:
  - computed_due_date, computed_unlock_date
  - is_blocked_by_dependency (bool)

8. KEY RPC FUNCTIONS
----------------------

RPC: hybrid_search_v2
  Params: (p_org_id UUID, p_pack_id UUID, p_query_text TEXT,
           p_query_embedding VECTOR(1536), p_match_threshold FLOAT DEFAULT 0.5,
           p_match_count INT DEFAULT 15, p_rrf_k INT DEFAULT 60,
           p_module_key TEXT DEFAULT NULL, p_track_key TEXT DEFAULT NULL)
  Returns: TABLE(id UUID, path TEXT, content TEXT, entity_type TEXT,
                 entity_name TEXT, signature TEXT, line_start INT, line_end INT,
                 score FLOAT)
  Algorithm: RRF (Reciprocal Rank Fusion) of pgvector + FTS results with
             optional 1-step graph expansion (imported→exported_names matching)
  Security: SECURITY DEFINER; filters is_redacted=false and active generation
  Privileges: Revoked from PUBLIC; called from service_role only

RPC: definition_search_v1
  Params: (p_org_id UUID, p_pack_id UUID, p_symbols TEXT[], p_match_count INT)
  Returns: TABLE of matching chunks where entity_name IN symbols
  Security: SECURITY DEFINER; used by detective loop (Hop 1)
  Purpose: Finds function/class definitions by name for multi-hop retrieval

RPC: purge_source_v1
  Params: (p_pack_id UUID, p_source_id UUID, p_actor_user_id UUID)
  Returns: JSONB { knowledge_chunks: N, ingestion_jobs: N }
  Algorithm:
    DELETE knowledge_chunks WHERE pack_id=? AND source_id=?
    DELETE ingestion_jobs WHERE pack_id=? AND source_id=?
    UPDATE pack_sources SET last_synced_at=NULL WHERE id=? AND pack_id=?
  Security: SECURITY DEFINER
  Called by: purge-source Edge Function (after RBAC check)

RPC: has_pack_access(user_id UUID, pack_id UUID, required_role TEXT)
  Returns: BOOLEAN
  Purpose: Reusable RLS helper. Checks pack_members.role >= required_role.
  Used by: RLS policies across lifecycle, audit, and roadmap tables.

RPC: is_pack_member(user_id UUID, pack_id UUID)
  Returns: BOOLEAN
  Purpose: Simpler membership check (any role). Used by playlist/content RLS.

9. MIGRATION CONVENTIONS
--------------------------
  - ALL migrations are in supabase/migrations/ with timestamp prefix YYYYMMDDHHMMSS_
  - NEVER edit an old migration. Always create a new file.
  - NEVER add USING(true) or WITH CHECK(true) for non-SELECT operations.
  - Use SECURITY DEFINER for RPCs; revoke PUBLIC execute; grant only service_role.
  - Always add IF NOT EXISTS for tables/indexes.
  - Use CREATE OR REPLACE for functions (idempotent).
  - Every new table must have RLS enabled (ALTER TABLE ... ENABLE ROW LEVEL SECURITY).

10. KEY INVARIANTS
-------------------
  - knowledge_chunks is_redacted=true chunks NEVER appear in hybrid_search_v2.
  - Purge is always source-scoped: WHERE pack_id=? AND source_id=? on all deletes.
  - ai_audit_events has NO UPDATE/DELETE policies (append-only by design).
  - legal_hold=true in pack_lifecycle_policies prevents retention cleanup.
  - Concurrency lock before writing new generation chunks is enforced via
    supabase/migrations/20260325000000_job_locks.sql.

11. HOW TO TEST
----------------
  1.  Apply all migrations on a fresh DB: supabase db reset
  2.  Verify hybrid_search_v2 returns redacted=false chunks only
  3.  Verify purge_source_v1 is idempotent (call twice, no error)
  4.  Verify has_pack_access('learner_uid', pack_id, 'author') returns false
  5.  Check supabase/qa/lifecycle/ for repeatable lifecycle RLS test scripts
